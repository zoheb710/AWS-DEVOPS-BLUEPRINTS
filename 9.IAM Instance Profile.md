# 👤 AWS IAM Instance Profile — Setup Guide

***

## 💡 What is an IAM Instance Profile?

An **Instance Profile** is a container that passes an **IAM Role** to an EC2 instance at launch. When you create a role via the AWS Console for EC2, AWS **automatically creates the instance profile** with the same name — you don't need to create them separately. ✅

***

## 🅐 Part A — Create the IAM Role

### 🔹 Step 1 — Open IAM Console
- Go to **AWS Console → IAM → Roles → Create Role**

### 🔹 Step 2 — Select Trusted Entity

1. Trusted entity type: **AWS Service**
2. Use case: **EC2**
3. Click **Next** ➡️

### 🔹 Step 3 — Attach Permission Policy 🔐

1. In the search box, type: `AmazonSSMManagedInstanceCore`
2. ✅ **Check** the policy `AmazonSSMManagedInstanceCore`
3. Click **Next** ➡️

> 🛡️ This policy allows EC2 to communicate with **AWS Systems Manager** — enabling **Session Manager** (browser-based shell) without SSH or a bastion host

### 🔹 Step 4 — Name & Create

1. 🏷️ Role name: `MySSMRole` *(or `EC2-SSM-Role`)*
2. Review trusted entity shows `ec2.amazonaws.com`
3. Click **Create Role** ✅

***

## 🅑 Part B — Attach to Launch Template *(For ASG — Recommended)*

> 💡 Since you are using an **Auto Scaling Group with a Launch Template**, attach the role here so every new EC2 launched by ASG automatically gets the profile

1. Go to **EC2 → Launch Templates** 📋
2. Select your launch template → **Actions → Modify Template (Create new version)**
3. Scroll to **Advanced Details** ⬇️
4. Under **IAM instance profile** → select `MySSMRole` from the dropdown
5. Click **Create Template Version** ✅
6. Go back to **Launch Templates → Actions → Set Default Version** → select the new version ✅

***

> 🐧 **Important for Ubuntu:** Ubuntu 22.04 LTS AMIs on AWS come with **SSM Agent pre-installed** — no manual installation needed. Just attaching the role is enough! 🎉
